### What is Civic Auth?

Civic Auth is a user-friendly authentication solution for Web3 applications. It enables users to log in via:

- ✅ **Email**
- ✅ **Google**
- ✅ **Passkeys**
- ✅ **Crypto Wallet** (including embedded wallets)

Civic also supports **embedded wallets**, which are automatically created for users without requiring browser extensions. This simplifies onboarding for decentralized apps.

### Civic Wallet Benefits

The Civic Wallet integration provides RostraFi users with:

- 🔒 **Self-Custodial Security**: Users maintain control of their assets
- 💸 **Seamless Transactions**: Easy sending and receiving of rewards
- 🌐 **Multi-Chain Support**: Manage assets across different blockchains
- 🪪 **Identity-Verified Accounts**: Reduced fraud and enhanced trust
- 👆 **Biometric Protection**: Additional security layer for sensitive operations
- 🏆 **Reward Management**: Direct deposit of competition winnings

## 🔄 Authentication Flow

<p align="center">
  <img src="https://github.com/user-attachments/assets/02d0c043-b508-4e2d-b6dd-1e08cfe33417" alt="Authentication Flow" width="600" height="400" />
</p>


Civic Auth provides a streamlined authentication flow:

1. **User Initiates Login**: Via email, Google, passkey, or existing wallet
2. **Identity Verification**: Optional verification steps based on application requirements
3. **Wallet Creation/Connection**: Automatic embedded wallet creation for new users or connection to existing wallets
4. **Session Establishment**: Secure session with appropriate permissions
5. **Ongoing Verification**: Periodic re-verification for sensitive operations

> ⚠️ Note: The term "Flow" in Civic Auth refers to the **authentication flow** (login process), **not** to the Flow blockchain.
